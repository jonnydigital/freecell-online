## CRITICAL BUGS (Fix Immediately)

### 1. Mobile Game Board Completely Blank
**Severity:** SHOWSTOPPER
**Where:** playfreecellonline.com + solitairestack.com — all game modes on mobile viewport
**What happens:** On mobile (tested 390x844), the game board shows only the green background. Cards exist in the DOM (52 elements, correct styles, opacity 1) but are invisible. The top bar (#1, timer, moves) and bottom nav (Menu, New, Undo, Redo, Hint) render fine.

**Root cause found:** In `src/components/dom-freecell/DomGameShell.tsx`:
- Line ~1006: Container `div.relative.flex-1.overflow-clip` collapses to **height: 0px** on mobile
- Line ~1011: Inner `div.absolute.inset-0.overflow-clip` becomes 16px tall inside 0px parent
- The parent at line ~740 uses `min-[1180px]:h-[calc(100dvh-1rem)]` which only applies at 1180px+
- On mobile, no explicit height → flex-1 collapses → absolute children invisible

**Fix:** Add explicit mobile height to the game container. Something like:
```
h-[calc(100dvh-6rem)] min-[1180px]:h-[calc(100dvh-1rem)]
```

### 2. Google AdSense Ads Not Rendering (Blank Ad Slots)
**Severity:** CRITICAL (blocking revenue)
**Where:** All pages — the ADVERTISEMENT / SPONSORED sidebar box is completely empty
**Evidence:**
- AdSense script loads (1 script tag found)
- 6 `<ins class="adsbygoogle">` elements on page
- ALL show `data-ad-status="unfilled"` — Google is not serving ads
- Console warning: "AdSense head tag doesn't support data-nscript attribute"

**Multiple issues found:**

**Issue A — Conflicting AdSense loading paths:**
- `layout.tsx` line 24-28: Loads AdSense via Next.js `<Script strategy="afterInteractive">`
- `CookieConsent.tsx` line 76-85: Also loads AdSense dynamically after cookie consent
- The Next.js Script adds `data-nscript` attribute which AdSense warns about

**Issue B — Cookie consent gating is bypassed:**
- `layout.tsx` loads AdSense unconditionally (no cookie check)
- `CookieConsent.tsx` tries to gate it behind consent, but layout loads it first
- This creates a race condition and the `data-nscript` warning

**Issue C — Likely low traffic / account issues:**
- Even with script loaded, all slots return "unfilled"
- This could mean: AdSense account not fully approved, ad review pending, or insufficient traffic for fill rate
- Verify at https://adsense.google.com that the account (ca-pub-3083538874906149) is active and approved

**Fix recommendations:**
1. Remove the duplicate `<Script>` tag from `layout.tsx` — let `CookieConsent.tsx` be the sole loader
2. OR: Remove the CookieConsent gating entirely and load AdSense in layout (simpler, competitor does this)
3. Use a regular `<script>` tag instead of Next.js `<Script>` to avoid the `data-nscript` attribute
4. Verify AdSense account status and ensure the site is added/approved in AdSense dashboard
5. Add specific ad slot IDs to all AdUnit instances (some have `slot` prop, some don't)

## HIGH-PRIORITY UX ISSUES

### 3. Game Dropdown Menu Doesn't Auto-Close
**Where:** Game selector dropdown (top-left "FreeCell" button)
**What:** Opening the dropdown, then clicking on the game board does NOT close the menu. The menu stays open and overlaps the game area. User can interact with cards through/behind the menu.
**Expected:** Clicking anywhere outside the dropdown should dismiss it.

### 4. Empty/Dead Leaderboard
**Where:** Sidebar — "DAILY LEADERBOARD" section
**What:** Shows "No scores yet today — Win today's challenge to claim #1" with a trophy icon. Looks empty and dead. No social proof.
**Competitor comparison:** online-solitaire.com shows actual player names, times, moves, and scores (e.g., "ze21 — 00:23, 68 moves, 91 score"). This creates real competitive engagement.
**Fix:** Either seed with bot data to make it feel alive, or if real user data exists, surface it. The empty state kills the competitive motivation.

### 5. Large Empty Gaps in Below-Fold Content
**Where:** Homepage below the game — between "Play FreeCell Online" intro text and "How to Play FreeCell" section
**What:** ~200px of empty green space with nothing in it. This is prime real estate for an ad unit or cross-promotion.
**Also:** Another large gap between FAQ section and "More Games" cards.
**Competitor:** online-solitaire.com has zero wasted space — content flows directly with ads interspersed.

### 6. No Monthly Challenge Calendar on playfreecellonline.com
**Where:** Sidebar
**What:** solitairestack.com has a March 2026 calendar in the sidebar, but playfreecellonline.com does not.
**Competitor:** online-solitaire.com has a beautiful monthly challenge with milestone badges (10 wins, 20 wins, 31 wins with trophy icons) and a progress bar. This is a major retention driver.
