## BUGS FOUND TODAY

### Bug #1 — Bristol Solitaire Layout Overflow (CRITICAL → FIXED ✅)
**Where:** playfreecellonline.com/bristol
**What:** The Bristol game renders 11 cascade columns (8 fans + 3 reserve piles) but the `--card-width` CSS variable in `dom-card-styles.css` is computed for 8 columns (`calc((100vw - 380px) / 8.6)`). With 11 columns, cards overflow both the left and right viewport edges — the leftmost fan column (4♣) is clipped, and the 3rd reserve pile is cut off.
**Root Cause:** `GenericCascadeBoard.tsx` imports `dom-card-styles.css` directly but had no mechanism to adapt card sizing for different column counts.
**Fix Applied:** Added `cardWidthOverride` useMemo in `GenericCascadeBoard.tsx` that activates when `cascades.length > 8`. Overrides `--card-width` to `clamp(60px, calc((100vw - 288px) / (n×1.08)), 100px)` where `n` is the cascade count. Verified the formula: at 1512px viewport with 11 cols → ~106px per card (within clamp max of 100px), all columns fit.
**Files Changed:** `src/components/dom-generic/GenericCascadeBoard.tsx`
**Commit:** `93d8d49` — pushed, Vercel deploy triggered.

### Bug #2 — Duplicate "Deal" in Game Page Titles (MEDIUM → FIXED ✅)
**Where:** playfreecellonline.com/game/1 (and any curated deal page)
**What:** Page title rendered as `"FreeCell Game #1 - First MS deal Deal"` — the word "Deal" appears twice. Label `"First MS deal"` already contains "deal", but the title template in `game/[number]/page.tsx` appended an additional ` Deal` suffix.
**Fix Applied:** Changed title template from `` `FreeCell Game #${gameNum} - ${deal.label} Deal` `` to `` `FreeCell Game #${gameNum} - ${deal.label}` ``. All curated deal labels are self-descriptive and don't need the suffix.
**Files Changed:** `src/app/(main)/game/[number]/page.tsx`
**Commit:** `93d8d49` — same commit as Bug #1.
