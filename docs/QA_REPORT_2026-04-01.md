## Phase 4: Improvements Shipped

### 1. Schema Consolidation — 9 Game Pages
**Commit:** `918aab6` — `seo: consolidate AggregateRating into WebApplication schema on 9 game pages`

**Problem:** bisley, cruel, la-belle-lucie, scorpion, seahaven, canfield, yukon, pyramid, and calculation each had two separate JSON-LD blocks for the same game: a `WebApplication` block without rating data, plus a separate `SoftwareApplication` block containing only the `aggregateRating`. This is non-standard and could cause schema conflicts or signal ambiguity to Google's rich results parser.

**Fix:** Moved `aggregateRating` directly into each page's existing `WebApplication` block and removed the standalone `SoftwareApplication` block. Now consistent with the pattern used by accordion, golf, klondike, and other game pages.

**Files changed (9):** bisley, calculation, canfield, cruel, la-belle-lucie, pyramid, scorpion, seahaven, yukon page.tsx files.

**Result:** 63 lines added, 197 lines deleted. Cleaner, standard schema structure across all game pages.

**Vercel deployment:** Building at time of report (push to main: `918aab6`). All previous deployments confirmed READY.
