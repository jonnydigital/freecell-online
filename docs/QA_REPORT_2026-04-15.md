### Bug Found & Fixed — JSON-LD Hydration Duplication

**Severity**: HIGH (SEO — structured data)

**Symptoms**: All `solitairestack.com` game variant pages (Pyramid, Golf, TriPeaks, Canfield, etc.) were showing **double the expected JSON-LD scripts** in the rendered DOM.

- Raw SSR HTML: 6 `<script type="application/ld+json">` tags (correct)
- Rendered DOM after hydration: 12 script tags (every schema duplicated)

**Root Cause**: React 18 hydration does not correctly reconcile `<script>` elements that use `dangerouslySetInnerHTML`. During client-side hydration, React re-inserts these scripts rather than reusing the server-rendered DOM nodes. This caused Google's structured data parser to see each schema twice, which can cause parsing warnings and reduced rich snippet eligibility.

Note: `playfreecellonline.com` was **not** affected because its SSR HTML count matched the DOM count exactly (5 scripts each). The issue was isolated to `solitairestack.com` game variant pages.

**Fix Applied** (commit `67224d3`):
Added a stable `id=` attribute to every `<script type="application/ld+json">` element across the codebase. React uses `id` to match server-rendered DOM nodes during hydration, preventing re-insertion.

- `JsonLd.tsx`: Updated component to auto-derive a stable `id` from `@type` + `name` (slugified). No changes needed at call sites for pages using this component.
- `app/layout.tsx`: Added `id="ld-organization"` to Organization script in `<head>`
- `(main)/layout.tsx`: Added `id="ld-webapplication-site"` to site-level WebApplication script
- `freecell/page.tsx`, `page.tsx` (all spoke homepages), `*BelowFold.tsx`: Explicit IDs added
- 13 additional game variant page files: IDs added via automated script

**Files changed**: 22 files, 131 insertions, 12 deletions

**TypeScript**: Clean (no errors)
