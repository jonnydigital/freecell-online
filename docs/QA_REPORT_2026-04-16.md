### solitairestack.com (HUB) — STALE / DEGRADED
Production HTML reflects the April 14 codebase. Issues currently live on production:
- No `id=` attributes on any of the 5 ld+json scripts → **hydration duplication (10 scripts in DOM)**.
- Hub `WebSite` schema description still the short `siteConfig.defaultDescription` fallback.
- No hub-level WebApplication + AggregateRating schema (the 4.8/5, 8372 ratings block added April 15).
- Hub `meta description` is still the developer-facing fallback.

All four of these are already fixed in `main` (commits 67224d3, 878d376 area) — but production deploys for solitairestack.com have been failing since commit `836e6bd`. Root cause (see next section): `SolitaireHubHome.tsx` has `'use client'` at the top and `export`s `featuredGames` and `faqItems`; the server component `src/app/(main)/page.tsx` imports and `.map()`s them during SSR of the hub, which throws on build:
```
TypeError: e.featuredGames.map is not a function
    at n (.next/server/chunks/ssr/[root-of-the-server]__5425554e._.js:1:13358)
    at stringify (<anonymous>)
Error occurred prerendering page "/"
Export encountered an error on /(main)/page: /, exiting the build.
```
**Suggested (not shipped):** hoist `featuredGames` and `faqItems` into a plain TypeScript module (e.g., `src/lib/hubContent.ts`) and import them from both the server page and the client `SolitaireHubHome` component. This change has **not** been made in this session — flagged for Jonathan's review because it touches the hub prerender path and should be confirmed and deployed while attended.
