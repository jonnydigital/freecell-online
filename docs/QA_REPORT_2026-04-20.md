## Mobile Investigation (unverified, iframe-proxied)

**Setup.** Loaded `/game/1` inside a 375×812 iframe on the parent page and measured via `getComputedStyle`/`getBoundingClientRect` on elements in the iframe document.

**Observations.**
- `.dom-card` elements: 52 present, `opacity: 1`, `display: block`, `position: absolute`, first card width 38.5 px (matches the April 16 CSS-arithmetic calculation of 39.25 px at 375 px viewport).
- `.dom-board-surface` element: `rectH: 210 px`, `rectY: 43` (inside the iframe), no overflow on itself.
- **Parent chain problem.** The board's grandparent (`relative flex-1 overflow-clip`, `DomGameShell.tsx:976`) reported `height: 0px`. Its parent (`flex h-full flex-col overflow-clip`, line 753) reported `height: 105px` even though ITS parent (`min-w-0 flex-1 h-[100dvh]` with inline style `height: calc(100dvh - 1rem)`, line 748) reported `rectH: 806`. If `h-full` on a child of an explicitly-sized parent really resolves to 105 px, that would be a mobile layout bug. But this behavior wasn't seen on real mobile in the April 16 CSS-arithmetic pass, so the likely explanation is iframe-specific dvh resolution.

**Why I didn't ship a fix.** Two hypotheses are live:
1. **Real bug:** the `h-full` child doesn't inherit the parent's explicit height because of how the sticky/dvh interplay lays out on mobile. A fix would add an explicit height fallback or change the inner wrapper to `flex-1 min-h-0`.
2. **Iframe artifact:** `calc(100dvh - 1rem)` inside an iframe hosted by a desktop browser doesn't resolve to the iframe's inner height the same way it would on a real phone. In that case the production code is fine and a fix would actually break real mobile.

Without a real phone or a working device-emulation path, I can't distinguish these. Please manually check `https://playfreecellonline.com/game/1` on an iPhone or Android Chrome. If you see the top bar + action bar + a huge empty green area (no cards), case 1 is live and we need a fix. If you see cards rendering normally, case 2 was the culprit and this can be dismissed.

**Repro for manual verification.**
1. Open `https://playfreecellonline.com/game/1` on a real mobile browser (≤ 414 px wide).
2. Observe whether cards render in the board region.
3. Expected: 52 cards visible across 8 cascade columns with free cells + foundations above.
4. Failure mode: top bar and action bar render, then empty green area down to the cookie banner.
