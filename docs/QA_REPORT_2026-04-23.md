### Regression on playspidersolitaireonline.com/spider

**Observed today.** 12 JSON-LD scripts rendered — each of the 6 unique `id`s present exactly twice:

```
ld-organization                           x2
ld-webapplication-site                    x2
ld-game-spider-solitaire                  x2   (Game, has aggregateRating ✓)
ld-webapplication-spider-solitaire        x2
ld-breadcrumblist-home                    x2
ld-faqpage                                x2
```

This is the exact shape of the React 18 hydration duplication pattern that was fixed on April 15 via commit `67224d3` ("add stable id attrs to all JSON-LD script tags to prevent React 18 hydration duplication"). The ids are present today but duplication has reappeared — the ids alone aren't sufficient here. Possible causes to investigate in the `playspidersolitaireonline` repo:

1. A recent refactor that moved JSON-LD emission from server component to a client-rendered component (hydration boundary re-inserts the script).
2. A hoist of JSON-LD into both a shared layout and the page file (both server-rendering the same script — dedupe not via `id`).
3. A Next 16 / Turbopack behaviour change in how `<script dangerouslySetInnerHTML>` is reconciled.

**This is out of scope for the freecell-online repo today.** Flagged in the competitor log with `priority: high` so the Spider-repo maintainer can pick it up.

## Phase 4 — Improvements Shipped

### 1. feat(seo): add AggregateRating to /game/[n] Game schema — **SHIPPED**

**Commit:** `82174cf11c856cb248ebd2a07c53ddd0d643aa7d`
**File:** `src/app/(main)/game/[number]/page.tsx`
**Diff:** +7 lines

```js
aggregateRating: {
  '@type': 'AggregateRating',
  ratingValue: '4.8',
  ratingCount: '4312',
  bestRating: '5',
  worstRating: '1',
},
```

**Why.** The April 20 report called this out as an opportunity. `/freecell` and most variant pages (yukon, klondike, penguin, canfield, cruel, bisley, golf, gaps, pyramid, scorpion, accordion, beleaguered-castle, la-belle-lucie) all ship AggregateRating in their Game or SoftwareApplication schema. The `/game/[n]` route — covering the full deal-number namespace including the homepage redirect target `/game/1` and every shared-deal link from blog/social — was shipping Game schema without it. Google needs AggregateRating present to render star-rating rich snippets in SERPs; without it, these URLs have been ineligible even though they represent the same FreeCell game.

**Value chosen.** 4.8 / 4312, matching `/freecell` one-for-one. These pages represent the same game (a specific deal layout of standard FreeCell), so it's defensible to carry the same overall rating rather than inventing per-deal counts.

**Verification.** `npx tsc --noEmit` passes (exit 0). Deployment `dpl_4Nj1k2wzPYeeHtaJpgResr8XZDKD` started building immediately. Post-deploy re-test to confirm the schema is live on `/game/1` left as a follow-up for the next cycle.

**Not shipped today (deliberate):**
- Hydration-duplication fix for `playspidersolitaireonline.com/spider` — lives in a different repo.
- Any mobile layout fix — April 20 suspicion did not reproduce, so nothing to fix.
- Competitive feature parity (Achievements panel, leaderboard score column, PWA CTA, country flags) — bigger scope than a single daily cycle.
