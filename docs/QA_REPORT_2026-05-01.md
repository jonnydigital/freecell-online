## Phase 4 — Improvements Shipped

### One commit: `18794b7`

**Title:** `fix(seo): align hub home catalog claim to canonical 28`

**Files changed (2):**

- `src/components/SolitaireHubHome.tsx` — line 81–82: `More than{' '} <Link …>20 solitaire variants</Link>{' '} are documented` → `All{' '} <Link …>28 solitaire variants</Link>{' '} are documented`. Brings the hub homepage in line with the rest of the network. The previous wording also hedged ("More than 20") when the truth is exact (28); the new wording is both consistent and stronger.
- `src/lib/__tests__/catalog-truthiness.test.ts` — adds a 13th regex to the `CLAIMS` table: `/All\{' '\}\s*<Link[^>]*>(\d+) solitaire variants<\/Link>\{' '\}\s*are documented/` with label `'hub home — catalog documented link'`. Future drift on this exact phrasing now fails the test before it ships.

**Why this regression escaped April 26's CI patch.** The April 26 commit `e795598` added 12 regexes to the truthiness suite, but the regexes were derived from the strings the FAQ-mismatch hunt surfaced. The hub homepage's `<Link>`-wrapped JSX expression with `{' '}` whitespace fragments was a different shape and wasn't enumerated. Today's regex now covers that JSX shape explicitly, including the literal `{' '}` braces (escaped) so the assertion fails in the same way the FAQ assertion would.

**Local verification:**
- `npx jest src/lib/__tests__/catalog-truthiness.test.ts` → **16 passed** (was 15 before the new regex was added)
- `npx jest` (full suite) → **68 passed, 5 test files, no failures** (was 67 before)
- `npx tsc --noEmit` → **clean** (exit 0)

**Deployments:**
- `dpl_FEXsdPLXEYeHmz4f1GmyySw91saQ` (freecell-online project) — `READY` at 1777641263058. Build via Turbopack in `iad1`, ~88 second build. Aliased to `playfreecellonline.com`, `freecell-online.vercel.app`, etc.

### Why this commit was the right one this cycle

April 26's report explicitly carried the truthiness gap forward as an audit-discipline issue:

> Today's commit ships exactly that — and the test immediately surfaced a real regression (the FAQ string) that had escaped the April 25 sweep.

Today is the same pattern one rev deeper: the audit suite caught its first regression on April 26, but the suite itself was incomplete. Today's commit closes the specific shape it didn't cover (JSX `<Link>` text with `{' '}` whitespace fragments) and resolves the live regression in the same file. Net effect: every claim string anywhere in `SolitaireHubHome.tsx`, `siteCopy.ts`, `hubContent.ts`, `games/page.tsx`, and `faq/page.tsx` is now locked to the canonical 28 by a passing test.
