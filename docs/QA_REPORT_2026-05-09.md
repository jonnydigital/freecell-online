### Carry-forward issues

- **Hub DOM hydration duplication** (since 2026-04-26): SSR HTML is clean (one
  script per `id`), but every `<JsonLd>` script gets duplicated at React-19
  hydration despite the stable `id` workaround. Affects every page that uses the
  `JsonLd` component (Pyramid / Golf / TriPeaks / Forty-Thieves / Beleaguered
  Castle / etc.). The `/freecell` page is the only spot-checked page that stays
  clean ‚Äî it uses `FreecellBelowFold` which embeds `faqJsonLd` via raw
  `dangerouslySetInnerHTML`, so the bug is specifically in the `JsonLd`
  component's render path. SSR HTML is what Google indexes so SEO is unaffected
  (Search Console hasn't flagged it). Fix vector unchanged: move JSON-LD into
  Next.js `metadata.other` or wrap in `next/script`. Cosmetic / code-quality.
- **playfreecellonline.com `/` canonical ‚Üí solitairestack.com/`** ‚Äî intentional
  per `PRD-klondike-spider-site-launch.md` (HUB_ONLY route ownership for AdSense
  duplicate-content protection); confirmed in source at
  `src/lib/routeOwnership.ts:58` (`'/': HUB_ONLY`) and the `canonicalUrlFor()`
  resolver. Documented as design, not a bug.
