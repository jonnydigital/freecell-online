inner viewport reported by JavaScript stays pinned at 1512 px even after the
resize call returns successfully. JavaScript `window.outerWidth` reports 0
post-resize while `innerWidth` stays at 1512. This matches the May 6 / May 9
findings — the Chrome MCP appears to render at fixed display resolution
independent of OS window size. True device-width interactive QA could not
run this cycle (carried forward limitation).

Mobile rendering verified three ways instead:

1. **CSS audit** — `src/components/dom-freecell/dom-card-styles.css` mobile
   media query at line 61–79 unchanged from May 9:

   ```css
   @media (max-width: 767px) {
     .dom-board-surface {
       --card-width: clamp(30px, calc((100vw - 61px) / 8), 54px);
       --card-height: calc(var(--card-width) * 1.4);
       --cascade-overlap: clamp(10px, calc(var(--card-width) * 0.32), 18px);
       --pile-gap: clamp(2px, 0.8vw, 5px);
       --board-padding-x: clamp(4px, 1.5vw, 10px);
       --board-padding-y: clamp(6px, 2vw, 12px);
       --board-max-width: 100%;
       /* + clamped corner-offset, rank-size, suit-size, slot-label-size */
     }
   }
   ```

   Plus the Spider override at `<=767px`:
   `--card-width: clamp(24px, calc((100vw - 66px) / 10), 44px)` for the
   10-column tableau (lines 104–110). Both intact, no regressions vs prior
   cycles.

2. **clamp() math** for the FreeCell mobile rule:
