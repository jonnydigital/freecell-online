### Mobile (375 / 390 / 414 px)

Interactive `resize_window` again floored at the macOS Chrome minimum (~755 px viewport, screen.availWidth 1512). Cannot trigger the mobile media query interactively in this environment.

Verified instead via CSS rule audit + clamp() math:
- `.dom-board-surface` mobile rule at `src/components/dom-freecell/dom-card-styles.css:61` unchanged: `--card-width: clamp(30px, calc((100vw - 61px) / 8), 54px)`.
- 375 px viewport → card-width = clamp(30, 39.25, 54) = **39.25 px** ✓
- 390 px viewport → card-width = clamp(30, 41.13, 54) = **41.13 px** ✓
- 414 px viewport → card-width = clamp(30, 44.13, 54) = **44.13 px** ✓
- Each within the 30–54 px target band. 8 cards × 41 px ≈ 328 px + 7 gaps × ~3 px + ~12 px board-padding ≈ 361 px, fits inside 375 px viewport.

Forced-clamp simulation on the live page: set the mobile clamp values directly on `.dom-board-surface` via JS. All 52 cards rendered at 39 px width × 54 px height, opacity 1, display block. No clipping confirmed below 375 px.

Mobile board render itself (the 2026-05-24 critical regression — flex-grow collapsing 100dvh shell height to 0 px below `lg`) remains fixed at commit `6fc3f3b` and was not regressed by today's commit (CSS untouched).
