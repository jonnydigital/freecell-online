### JSON-LD audit (1 known issue, 1 backlog confirmed)

| Page | SSR HTML | Runtime DOM | Status |
|------|----------|-------------|--------|
| `playfreecellonline.com/game/1` | 3 | 3 | clean |
| `solitairestack.com/freecell` | 5 | 5 | clean |
| `solitairestack.com/strategy` | 5 | 5 | clean |
| `solitairestack.com/spider` (→ redirects to playspidersolitaireonline.com/spider) | 6 | **12** | duplicated 2× |
| `solitairestack.com/golf` | 6 | **12** | duplicated 2× |
| `solitairestack.com/pyramid` | 5 | **10** | duplicated 2× |

**Root cause:** the affected pages render `<script type="application/ld+json">` inline in JSX `<body>`. Even when the page uses the `JsonLd` component (`src/components/content/JsonLd.tsx`) — which assigns a stable `id` so React hydration can reconcile — Next.js 15 App Router's streaming SSR + client hydration still inserts a second copy in BODY on these specific pages. The deployed `/freecell` and the per-game `/game/[id]` pages are unaffected, suggesting the difference is in how the page renders the schemas relative to client component boundaries.

**Impact:** moderate. Google Search Central tolerates duplicate JSON-LD with identical content, and the second copy is byte-for-byte identical, so this is unlikely to trigger a manual action. But it's wasteful, slightly increases page weight, and may confuse Search Console URL inspection.

**Fix path (deferred to architectural backlog):** move the `<script>` emission either (a) to `metadata.other` so Next.js renders it once in `<head>`, or (b) to `next/script` with `strategy="beforeInteractive"`. Either path is too broad for a single QA cycle commit — would touch every game route's `page.tsx`.
