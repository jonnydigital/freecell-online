### Architectural backlog

The duplication root cause is the same as documented June 1/2: `<script type="application/ld+json">` placed inline in JSX body of a server component that has a `'use client'` sibling tree. React hydration inserts a second copy in BODY even with stable id attributes. Fix path remains: migrate to Next.js metadata API `metadata.other` or `next/script` with `strategy="beforeInteractive"`. Lives in playspidersolitaireonline and playklondikeonline repos.
