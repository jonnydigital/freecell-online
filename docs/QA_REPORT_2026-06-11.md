## Phase 1 — Visual QA

### ⭐ Environment breakthrough: true mobile viewports finally captured

Five consecutive prior reports (Jun 3–9) carried the limitation "cannot render a true sub-500px mobile viewport." This run **solves it**: a CDP device-emulation harness (`tmp/cdp-mobile-shots.js`, plain Node 18 + `next/dist/compiled/ws`, no new deps) launches headless Chrome with `--remote-debugging-port`, applies `Emulation.setDeviceMetricsOverride` (mobile=true, real DPR), and captures pixel-true screenshots. First-ever real 375/390/414/768 renders of production in this project's QA history.

**Trap documented for future runs:** plain `--headless=new --window-size=375,812 --screenshot` is NOT a valid mobile test. Chrome lays the page out at its ~500px desktop floor and crops the PNG to 375 — producing a *false* overflow: cards hit the 54px clamp ceiling ((500−61)/8 = 54.9), board ≈ 500px, columns 7–8 "clipped", and the 448px (`max-w-md`) tutorial modal "overflows". All artifact, fully explained by 500px layout math (8×58−4+40 = 500 exactly). Initial screenshots this run showed exactly that and were re-taken with proper emulation before any code was touched. **No fix was needed and none was made.**
