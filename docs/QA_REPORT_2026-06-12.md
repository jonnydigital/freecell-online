## 4. Bug found, fixed, shipped

**Duplicate viewport meta dropped `viewport-fit=cover` (all spoke domains).** Root layout hand-wrote `<meta name="viewport" … viewport-fit=cover>`; Next.js (no `viewport` export present) auto-injected its default `width=device-width, initial-scale=1` *after* it. Last tag wins → `viewport-fit=cover` silently dead → `safe-area-inset-*` CSS inert on notched iPhones (home-indicator overlap risk for the fixed bottom bar).

- Fix: `export const viewport: Viewport = { width, initialScale, viewportFit: 'cover', themeColor }`; removed manual viewport + theme-color tags.
