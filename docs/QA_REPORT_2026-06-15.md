## 4. Improvements shipped (2)

### (1) Author page meta descriptions — commit `a702e9a`
The six author/desk profile pages used `bioShort` (254–371c) as their meta description, so search snippets were heavily truncated. Added an optional `metaDescription?: string` field to `src/lib/authors.ts` and a fallback in `authors/[slug]/page.tsx` (`author.metaDescription ?? author.bioShort`). Wrote concise ≤160c metas for all six desks (Editorial Team, Strategy, History, Rules, Research, J. Foye). **Visible bios unchanged** — only the meta/OG tag changed. The Person JSON-LD intentionally keeps `bioFull` (full text is fine for structured data).

### (2) Blog meta description override — commit `969b635`
Blog `description` doubles as the visible hero subtitle, so simply trimming it would degrade on-page content. Added an optional `metaDescription` frontmatter field to the blog pipeline (`src/lib/blog.ts` `BlogPost` + `blog/[slug]/page.tsx` `generateMetadata`), used **only** for the meta/OG tags — the hero subtitle still renders the longer `description`. Applied concise ≤160c metas to the **10 worst-truncated posts** (183–208c): klondike-solitaire-complete-guide, solitaire-history-from-napoleon-to-windows, klondike-vs-freecell, spider-solitaire-tips-for-beginners, what-your-favorite-solitaire-game-says-about-you, best-solitaire-games-for-beginners, spider-solitaire-difficulty-guide, best-solitaire-games-to-play-at-work, solitaire-variations-ranked-by-difficulty, spider-solitaire-on-windows-98.
