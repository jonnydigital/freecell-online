### Mobile — PASS (via iframe rig)
The live `resize_window` path was again unusable this run: the window's outer frame resized but the render viewport stayed pinned at `innerWidth` 1481 (> `outerWidth`), so the page never dropped below desktop width and mobile media queries never fired. Used the documented **same-origin iframe rig** (load the site inside a fixed-width iframe so its media queries respond to the iframe width; readable because same-origin). Responsive breakpoints confirmed at 767/768/1023/1279px; viewport meta correct (`width=device-width, initial-scale=1, viewport-fit=cover`).

> Note: solitairestack.com cannot be iframe-rigged directly — its game routes 301-redirect to spoke domains (e.g. /spider → playspidersolitaireonline.com), making the iframe cross-origin and unreadable. The rig was run on the same-origin spoke domains instead. Both sites share one codebase and the same responsive game components, so the playfreecellonline results validate the shared rendering path.
