### 🐛 Bug found — suit pip clipped on every buried card (mobile portrait, FreeCell)

Yesterday's run raised `--cascade-overlap` to `card-width × 0.42` to spend some of the
69% portrait dead space. That helped, but it was tuned against the **rank** glyph (8px)
rather than the **whole corner block**. Measured today on live prod:

| Viewport | `.dom-card__corner--tl` needs | Strip revealed | Suit pip clipped by |
|---|---|---|---|
| 375 | 20.1px | 16.3px | **3.8px** |
| 390 | 20.2px | 16.3px | **3.9px** |
| 414 | 20.4px | 18.3px | **2.1px** |
| 768 (portrait) | 23.5px | 18.3px | **5.2px** |

The corner block is `rank (8px) + suit-small (8.8px)` offset 3.3px from the card top, so it
runs to ~20.1px — but only 16.3px of each buried card was showing. Result: **every buried
card in every column displayed its rank with the suit symbol sliced off**, and suit is
exactly what a FreeCell player reads to plan alternating-colour moves. Fixed — see Phase 4.
