### 🐛 Bug found and fixed

`/freecell-vs-eight-off` emitted **two** `WebApplication` entities where its siblings emit one. The
extra was a page-level entity — `name: "FreeCell — Play Online Free"`, `url: <site root>` — a
copy-paste leftover from the game-page template. On an editorial comparison page that is wrong
twice over: it types the page as a game application, and it points at the homepage rather than at
the page itself, competing with the genuine site-wide `WebApplication` entity.

Auditing the whole family showed the same leftover on `/freecell-vs-bakers-game`. The other four
(`freecell-vs-spider`, `freecell-vs-klondike`, `klondike-vs-pyramid`, `spider-vs-scorpion`) were
already clean.

**Fixed** in `99958a6` — stray entity removed from both pages; they now emit
Article + FAQPage + BreadcrumbList like the rest of the family. `absoluteUrl` remains used in both
files, so no orphaned import. Verified: `npx tsc --noEmit` clean, `npm run lint` 0 errors.
