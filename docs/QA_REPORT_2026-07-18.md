### Finding — new panel shipped 2px under the project's own tap-target floor

The `MobileNextActionPanel` that landed 07-17 (`e9f669c`) renders correctly on
live at 375px (verified in a real device-emulated screenshot: title "Start with a
safe move", body copy, and Hint / Undo / Strategy buttons, with Undo correctly
disabled at 0 moves).

But it moved freecell's `smallTapTargetCount` from **4 → 6**. The two new entries
are the panel's own buttons:

```
{"text":"Hint",     "tag":"button","rect":{"width":104.33,"height":42}}
{"text":"Strategy", "tag":"a",     "rect":{"width":104.34,"height":42}}
```

42px is below `COMFORTABLE_TAP_TARGET_PX = 44` — the threshold the repo's own
audit harness asserts. It is above the hard-failure floor
(`MIN_TAP_TARGET_EDGE_PX = 32`), which is why the run stayed green: this is a
comfort regression, not a breakage. Fixed in Phase 4.
