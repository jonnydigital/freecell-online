## Phase 4 — Improvement shipped (1 slice)

Bucket priority: scorecard fails (none) → today's bugs (none) → **U6 play-UX
backlog** → competitor gaps. Highest non-empty = U6.

**Fix: removed duplicate Hint/Undo controls from the FreeCell mobile next-action
panel** (`src/components/dom-freecell/DomGameShell.tsx`).

- **Defect (logged 07-18 competitor-log):** in portrait mobile the persistent
  Mobile Bottom Bar already carries Menu / New / **Undo** / Redo / **Hint**,
  while the coaching next-action panel *also* rendered Hint + Undo + Strategy —
  so Undo (and Hint) appeared twice on screen simultaneously.
- **Change:** dropped `onHint`, `onUndo`, `canUndo` from the panel invocation.
  The panel now shows only its contextual coaching copy + the **Strategy**
  deep-link — its genuinely differentiated value. The persistent bottom bar is
  untouched, so no gameplay control is lost.
- **Scope discipline:** the same panel is used by the Spider, Klondike, and
  Generic shells, but they were **deliberately not touched** this run:
  Spider/Klondike need live mobile confirmation of their bottom-bar dup (deferred
  to backlog), and the **Generic shell has no persistent mobile bottom bar** —
  its panel is the *only* mobile control surface, so removing Hint/Undo there
  would be a regression. One small, fully-mapped slice (FreeCell, the primary QA
  target) only.
- **Validation:** `npx tsc --noEmit` ✅, `npm run lint` ✅ (0 errors, still 444
  warnings — none added), `npx jest` ✅ (72/72). `handleHint`/`handleUndo`/
