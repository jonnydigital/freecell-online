## Phase 4 — Improvements Shipped

**U6 slice — remove duplicate Hint/Undo from the Spider & Klondike mobile
next-action panels (completes the 07-19 FreeCell fix).**

Priority buckets 1 (scorecard ❌) and 2 (bugs found today) were both empty —
the network is green and today's QA found no defects — so the highest non-empty
bucket is a small U6 play-UX slice.

The 07-19 FreeCell fix (`ff5480b`) removed the redundant Hint/Undo from that
board's next-action panel because the persistent mobile bottom bar already
carries both, and it **explicitly deferred the Spider and Klondike shells**
pending live confirmation. Today I confirmed the same structure on both spokes:
each renders a `fixed inset-x-0 bottom-0 … sm:hidden` bottom action bar
(Home/New/Undo/Hint) **and** a `MobileNextActionPanel` that was still passing
`onHint`/`onUndo`, producing a second on-screen Hint + Undo on portrait phones.

Fix: dropped `onHint`/`onUndo`/`canUndo` from both panels, leaving the
contextual coaching copy + the editorial deep-link (Spider → Tips, Klondike →
Strategy) — mirroring the FreeCell panel exactly. `handleHint`/`handleUndo`
remain wired to each shell's top toolbar, bottom bar, and keyboard shortcuts, so
no unused-symbol regressions. Files:
`src/components/dom-spider/DomSpiderShell.tsx`,
`src/components/dom-klondike/DomKlondikeShell.tsx`.

Gate after change: `npx tsc --noEmit` ✅, `npm run lint` ✅ (0 errors, 444
warnings). The Generic shell is intentionally left as-is (its panel is the sole
mobile control surface — removing its buttons would strip controls).
