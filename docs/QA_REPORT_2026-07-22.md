## Phase 4 — Improvements

No ❌ scorecard checks and no Phase 1–3 bugs, so the highest non-empty bucket is
the **U6 play-UX / a11y backlog**. Took one small slice from the 07-21 backlog:

**`fix(a11y): expose FreeCell landscape status pill to assistive tech`**
(`src/components/dom-freecell/DomGameShell.tsx`)

- The landscape-mobile floating status pill (#game / timer / moves) carried
  `aria-hidden="true"`, hiding the game's only landscape status readout from
  screen readers — while the **portrait** top bar exposes the identical
  timer/moves as plain readable spans. Neither uses `aria-live`, so there was no
  timer-announcement-spam reason for the hide; it was simply an a11y parity gap
  introduced alongside the landscape feature (`78d2721`).
- Removed the `aria-hidden` and added a localized `aria-label` on the timer span
  (`{copy.time}: mm:ss`, present in all 6 locales) so the terse value reads
  coherently. The moves span is already self-describing (`N {movesShort}`).
- **Layout-neutral** — no CSS/geometry change, so no landscape visual tooling was
  needed to verify; the change is confined to the accessibility tree and is
  covered by the tsc + lint gate. Restores parity with the shipped portrait
  treatment.
